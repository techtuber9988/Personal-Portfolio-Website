# Project Structure Plan

## Folder Structure Ideas

- **src/**
  - `components/` – reusable UI parts (header, footer, project-card, etc.)
  - `pages/` – top‑level page components (Home, About, Projects, Contact)
  - `styles/` – global CSS/SCSS, variables, mixins, and page‑specific styles
  - `assets/` – fonts, icons, and any non‑image media
  - `utils/` – helper functions, API clients, validation logic
  - `hooks/` – custom React or Vue hooks if using a framework
  - `tests/` – unit and integration test files that mirror `src` structure

- **public/** (or `dist/` if you build to there)
  - `index.html` – the main entry point (may also be generated by a build tool)
  - `images/` – profile pictures, background photos, etc.
  - `favicon.ico`, `robots.txt`, `manifest.webmanifest`

- `assets/` (top‑level) – static files that need to be copied verbatim, e.g., `robots.txt`, `humans.txt`

- `scripts/` – build, lint, and deployment scripts (GitHub Actions, Azure Pipelines, etc.)

- `config/` – environment configuration (`.env`, `webpack.config.js`, `vite.config.ts`, etc.)

- `docs/` – project documentation or milestone descriptions

- `tests/` – end‑to‑end tests (Cypress, Playwright) if applicable

- `requirements.txt` – list of Python or Node dependencies (e.g., Pillow, Flask, or `react`, `next`, etc.)

- `README.md` – entry point for contributors; describe commands like `npm run dev` or `python -m http.server`

---

## Routing / Page Plan

| Path | Page | Purpose |
|------|------|---------|
| `/` | Home | Portfolio landing, quick intro |
| `/about` | About | Personal bio, skillset, resume link |
| `/projects` | Projects | Showcase with filtering by tech |
| `/contact` | Contact | Form, social links, email |
| `/blog` | (Optional) Blog | Articles or updates |

---

## Tech Stack Suggestions

- **Static Site Generators** – Next.js, Nuxt.js, Gatsby, Hugo, Jekyll, Eleventy.
- **Framework** – plain HTML/CSS/JS or React/Vue/Svelte with a bundler like Vite or Rollup.
- **Styling** – TailwindCSS, SCSS, or a CSS‑in‑JS solution if using a framework.
- **Testing** – unit tests with Jest/AVA and integration tests via Cypress or Playwright.
- **CI/CD** – GitHub Actions to lint, build, and deploy to platforms such as Netlify, Vercel, or GitHub Pages.

## Folder‑level Split Example

```
my-portfolio/
├─ src/
│  ├─ components/
│  ├─ pages/
│  ├─ styles/
│  ├─ assets/
│  └─ utils/
├─ public/
│  ├─ images/
│  └─ index.html
├─ scripts/
├─ config/
├─ docs/
├─ tests/
├─ requirements.txt
└─ README.md
```

This structure keeps the codebase modular, testable, and ready for future growth.
